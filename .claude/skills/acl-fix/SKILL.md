# Fix ACL Permission Errors for Tauri Plugin Commands

This skill fixes ACL errors like `plugin-name|command_name not allowed by ACL`. First ensure the app enables existing permissions from the plugin's ACL manifests. Only add new permissions to the plugin when the command truly isn't defined.

## Context

Tauri's Access Control List (ACL) system requires explicit permission definitions for each command that can be called from the frontend. New commands won't work until permissions are properly configured.

## Steps

### 1. Identify the command

Extract `plugin-name|command_name` from `$ARGUMENTS` or the error message.

Example:
`plugin:android-mediastore|request_permissions`

Convert snake_case commands to permission naming convention:
- `file_reader_open` → `allow-file-reader-open` / `deny-file-reader-open`

### 2. Check if the command already exists in plugin ACL manifests (preferred)

Look for the command in the plugin's `permissions` manifests:
- `permissions/autogenerated/acl-manifests.json`
- `permissions/default.toml`
- `permissions/autogenerated/commands/*.toml`

If the command exists, **do not create new permissions**. Instead, enable the plugin's permission in the app's capabilities:

```json
// src-tauri/capabilities/default.json
{
  "permissions": [
    "core:default",
    "plugin-name:default",
    "plugin-name:allow-command-name"
  ]
}
```

Use the exact permission identifier from the plugin's ACL manifests (e.g. `android-mediastore:allow-request-permissions`).

### 3. Create Permission File (only if command is missing)

Create a TOML file in `permissions/autogenerated/commands/[command_name].toml`:

```toml
# Automatically generated - DO NOT EDIT!

"$schema" = "../../schemas/schema.json"

[[permission]]
identifier = "allow-command-name"
description = "Enables the command_name command without any pre-configured scope."
commands.allow = ["command_name"]

[[permission]]
identifier = "deny-command-name"
description = "Denies the command_name command without any pre-configured scope."
commands.deny = ["command_name"]
```

Replace `command-name` (with hyphens) and `command_name` (with underscores) accordingly.

### 4. Update Default Permissions

Add the new allow permission to `permissions/default.toml` in the permissions array:

```toml
[default]
description = "Default permissions for the plugin"
permissions = [
  "allow-existing-command",
  "allow-command-name",  # Add this
]
```

### 5. Update Schema

Add two permission kind entries to `permissions/schemas/schema.json` in the `PermissionKind.oneOf` array:

```json
{
  "description": "Enables the command_name command without any pre-configured scope.",
  "type": "string",
  "const": "allow-command-name",
  "markdownDescription": "Enables the command_name command without any pre-configured scope."
},
{
  "description": "Denies the command_name command without any pre-configured scope.",
  "type": "string",
  "const": "deny-command-name",
  "markdownDescription": "Denies the command_name command without any pre-configured scope."
}
```

### 6. Verify and Rebuild

After making changes:
- Rebuild the plugin
- Rebuild the app
- Test the command

## File Structure Reference

```
permissions/
├── default.toml                 # Default permission set
├── schemas/
│   └── schema.json              # Permission definitions (PermissionKind.oneOf array)
└── autogenerated/
    ├── acl-manifests.json       # Aggregated permission identifiers
    └── commands/
        ├── existing_command.toml
        └── new_command.toml     # Add new command files here
```

## Example Usage

```
/acl-fix plugin:android-mediastore|request_permissions
```

If `request_permissions` exists in the plugin ACL manifests:
- Update the app capability to include `android-mediastore:allow-request-permissions`.

Only if the command is missing:
- Create `permissions/autogenerated/commands/request_permissions.toml`
- Add `allow-request-permissions` to `default.toml`
- Add schema entries for `allow-request-permissions` and `deny-request-permissions`
